AppStaticPrefs

class AppStaticPrefs

This is a very simple “persistent user prefs” class. It is instantiated as a SINGLETON, and provides a simple, property-oriented gateway to the simple persistent user prefs in iOS. It shouldn’t be used for really big, important prefs, but is ideal for the basic “settings” type of prefs most users set in their “gear” screen.

  • updateUserForRootURI(_:inUser:inPassword:)

    This method allows us to add a login ID and password to be saved. The login ID is keyed to the URL, so you could have the same login for different servers. The password is only saved if we support TouchID. If the password is nil or blank, then the login is stored, but the password is removed from the keychain.

    Declaration

    inUser

    inPassword

    Return Value
